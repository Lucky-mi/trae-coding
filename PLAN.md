# 单词测评软件实现计划（待确认）

本文档用于在开发前对需求、架构、数据、算法、页面与交付物做一次可执行拆解。你确认后将按本文档开始搭建项目与实现功能。

## 1. 目标与范围

### 1.1 产品目标
- 面向学生的单词测评与复测产品：完成测评、给出水平评估、提供错题复测、可视化展示学习进展
- 同时提供后台管理系统：词库与题目规则维护、用户与数据统计、运行监控
- 视觉目标：现代化、动效丰富、主题化皮肤（深色/浅色/赛博等），整体观感显著优于传统测评站点

### 1.2 里程碑（交付分期，但最终覆盖全部功能）
- P0（可跑通核心测评闭环）：词库解析 → 出题 → 作答 → 结束判定 → 结果报告（本地匿名）
- P1（账号与云端）：注册登录、历史记录云同步、多端一致
- P2（自适应与复测）：动态难度、错题标记、重复测试策略
- P3（后台管理）：词库/规则编辑、统计看板、监控
- P4（特色扩展）：3D/成就/分享/排行榜/学习小组

## 3. 数据准备与词库规范

根目录存在 3 个 xlsx（小学/初中/高中）。在正式编码前，需要先“抽取并固化词库字段规范”，避免后续算法与管理端返工。

### 3.1 xlsx 解析目标（统一字段）
为兼容多套词表，先定义内部统一模型（字段缺失可为空）：
- word：单词本体（主键候选）
- stage：学段（小学/初中/高中）
- level：级别（如 小学1级…高中3级）
- unit/book：册/单元（若词表提供）
- pos：词性（n/v/adj/adv…）
- lemma：词元（可选）
- forms：词形变化（复数/过去式/ing/比较级…，可选）
- phonetic：音标（可选）
- meaning_zh：中文释义（可选）
- example：例句（可选）
- tags：主题/场景标签（可选）

### 3.2 数据存储
- 本地（前端匿名）：IndexedDB（优先）或 LocalStorage（仅存轻量结果）
- 云端：关系型表 + 索引
- 词库版本：每次导入生成 version_id，测评记录引用版本，保证历史可复现

## 4. 测评规则与算法（需你确认的“规则口径”）

你提到“严格遵循提供的测评规则和水平评估标准”，但目前未看到规则文本。为确保实现一致，建议把规则做成可配置的“规则集”，由后台维护。

### 4.1 题目分布算法（示例口径）
你给的示例：“小学1级4个...高中3级4个”倾向于固定配额出题。
- 设定维度：stage + level
- 设定配额：每个 level 固定 N 题（默认 N=4，可配置）
- 抽题方式：
  - 词库去重（word 唯一）
  - 同一 session 内避免重复（包括复测策略中的重复需显式标记）
  - 抽样可按“词频/随机/分层”策略配置

需要你确认：
- level 列表与顺序：小学(1..?) 初中(1..?) 高中(1..3?) 是否都为 3 级？
- 每级题量是否统一为 4？是否存在某级更高权重？
- 是否要混排出题（跨 level 打乱）还是按 level 分段推进？

### 4.2 题型与交互
基础题型（P0 必须）：
- 四选一：选出正确释义 / 选出正确单词
可扩展（P2/P3+）：
- 拼写（听写/看中文拼写）
- 词形变化选择
- 例句填空

### 4.3 智能干扰项生成（规则化 + 可升级）
目标：干扰项“合理但不离谱”，同时避免与正确答案过近导致歧义。

候选策略（可组合加权）：
- 词性一致：优先同 pos
- 词形相似：编辑距离/前后缀相似（如 -tion/-sion、re-/un-）
- 长度相近：字符长度差 <= 2
- 语义近邻（可选）：使用嵌入向量或词表同主题 tags（若有）
- 释义相似（可选）：中文释义关键词重叠（若提供 meaning_zh）

排除规则：
- 与正确答案相同或大小写/变形等同
- 同一题的干扰项互相重复
- 干扰项曾在当前 session 作为正确答案出现（可配置）

### 4.4 动态难度调整（自适应）
建议使用“分段 + 置信度”模型，保证可解释性：
- 初始从中位 level 开始
- 连续答对提升 level，连续答错下降 level
- 引入置信度阈值：当对某个 level 区间的正确率稳定到一定程度后可提前结束

### 4.5 三种结束条件（需要你提供口径或确认默认）
默认建议（可配置，且后台可编辑阈值）：
1) 达到最大题量：完成所有配额题
2) 时间结束：倒计时为 0（session 直接结束并记未答为错/空）
3) 评估收敛：当前水平置信度达到阈值（例如 95%）或连续 K 题稳定在同一水平

### 4.6 错题标记与重复测试
- 错题集：按 user_id（或匿名 device_id）维度维护
- 复测策略：
  - 间隔复习：错题在 T+1/T+3/T+7 天再出现（可配置）
  - 相似题复测：错题的同词根/同词形/同主题扩展（如果词库能关联）

### 4.7 发音测评（Web Speech API）
- 功能：用户跟读 → 获取识别文本 → 与目标单词/短语做相似度 → 输出得分与纠错提示
- 注意：不同浏览器支持差异较大，需降级方案（不支持则隐藏入口）

## 5. 用户系统设计

### 5.1 匿名用户
- 识别：生成 device_id 存本地
- 数据：测评记录、错题集、成就进度
- 迁移：注册后可“绑定并上传”匿名历史到账号

### 5.2 注册用户
- 登录：邮箱/手机号 + 密码（或仅邮箱）与验证码（可选）
- 云同步：记录与错题、成就与学习组信息

### 5.3 水平评估可视化
- 雷达图：分维度能力（如 各学段/各 level 的掌握度、词性掌握度）
- 柱状图：本次测评各 level 正确率与耗时
- 时间轴：历史测评分数/水平变化

## 6. 前端页面与交互（信息架构）

### 6.1 用户侧页面
- Landing：产品介绍、开始测评、登录入口、主题切换
- 测评设置：选择测评范围（学段/级别/题量/是否自适应/是否限时）
- 测评进行：题目卡片、倒计时、进度条、动效切换、暂停（可配置）
- 结果页：总体水平、分布图表、错题列表、分享卡片
- 错题本：筛选/复测/发音练习
- 成就：徽章墙、升级动效
- 排行榜：全站/好友/学习组（登录后）
- 学习组：创建/加入/成员排行/组内挑战

### 6.2 管理端页面
- 登录与权限（admin）
- 用户管理：CRUD + 账号状态
- 词库管理：导入 xlsx、导出、编辑、版本回滚
- 规则管理：题量配额、结束条件阈值、自适应参数、干扰项策略开关
- 数据看板：DAU、测评次数、完成率、正确率分布、top 错词
- 监控：接口耗时、错误率、队列积压（如有）
